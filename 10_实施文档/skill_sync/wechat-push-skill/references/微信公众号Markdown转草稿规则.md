# 微信公众号 Markdown 转草稿规则

## 输入优先级

1. 优先使用 `正文结构与配图映射.md`
2. 再读取 `草稿箱正文.txt`
3. 再读取 `发布稿.md`

## 转换目标

要得到结构化草稿，不是逐行照搬：

- `title`
- `headings`
- `paragraphs`
- `cover_path`
- `image_map`

## 标题层级规则

- 一级标题只保留文章主标题
- 二级标题必须独立成段
- 三级标题必须独立成段
- 不允许把二级标题和三级标题吞进正文

## 正文规则

- 正文按自然段处理
- 不制造“正文段 + 空白段 + 正文段”的假排版
- 不要把所有图片先插到最上面再人工挪

## 图片规则

- 封面只认封面图
- 正文图只按 `正文结构与配图映射.md` 插入
- 每张正文图必须插在指定二级标题下的指定段落后
- 不允许编辑器侧临时猜测图片位置
# 封面补充规则

- 如果稿包中存在 `metadata.json`，封面优先读取 `cover_path`
- 如果同目录下已经有 `01_公众号封面_标题版_2.35比1.png`，则不允许回退使用旧竖版封面名
- 推送前必须核对封面文件与 `正文结构与配图映射.md` 中图 01 的文件名一致
