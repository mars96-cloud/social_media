# 本地成品读取与转换总则

## 目标

`wechat-push-skill` 不负责写内容，只负责把本地成品无损、严格地推到微信公众号草稿箱。

## 本地目录读取优先级

如果用户给的是一个发布目录，按下面优先级读取：

1. `正文结构与配图映射.md`
2. `草稿箱正文.txt`
3. `发布稿.md`
4. `images/`
5. `配图方案.md`

## 严格转换原则

1. 不允许原始 markdown 直接进入平台编辑器
2. 不允许缺少标题层级
3. 不允许缺少图片落点映射
4. 不允许封面与正文图混用
5. 不允许把所有正文图堆在文章最前面

## 推送执行原则

- 封面先按单独封面逻辑处理
- 封面处理必须优先读取 `metadata.json.cover_path`，并与 `正文结构与配图映射.md` 的封面文件保持一致
- 如本地同时存在 `2.35:1` 主封面和旧竖版封面，推送层不得自行择一兼容，必须只认主封面
- 正文后按结构化段落写入
- 再按映射逐张插图
- 最后保存草稿
