# WeChat Push Skill

## Scope

This skill only handles WeChat Official Account draft staging from local workspace files.

It does not:

- choose topics
- write new content from scratch
- rewrite the article strategy
- generate Xiaohongshu content
- publish live

Its job is narrow and strict:

1. read the local finished package
2. convert it into stable WeChat editor blocks
3. insert title, headings, body, cover, and mapped supporting images
4. save as draft only

## Input priority

When a folder is provided, read in this order:

1. `正文结构与配图映射.md`
2. `草稿箱正文.txt`
3. `发布稿.md`
4. `images/`
5. `配图方案.md`
6. `metadata.json`

If `正文结构与配图映射.md` is missing, do not guess image placement.

## Hard rules

- Never paste raw Markdown into the editor.
- Convert local content into structured editor blocks first.
- Keep major headings and prompt-style subheadings visible inside the WeChat editor.
- Merge broken one-line fragments into natural paragraphs before insertion.
- Body images must be inserted by the explicit mapping file, not by guesswork.
- Use the dedicated local cover image in this strict priority:
  1. `metadata.json.cover_path`
  2. `01_公众号封面_标题版_2.35比1.png`
  3. `01_公众号封面_标题版_2.35比1.jpg` / `.jpeg`
  4. `01_公众号封面.png`
  5. `01_公众号封面_标题版.png`
- Never lazily use the first body image as the cover.
- Never dump all body images at the top of the article.
- Only save to drafts. Do not click publish.
- Prefer stable local conversion over clever editor-side improvisation.
- If the package still points to a vertical cover or legacy cover filename while a `2.35:1` cover exists, stop and report the package as inconsistent instead of guessing.

## Heading rules

- The main article title is the only level-1 title.
- Level-2 headings must remain visible as section dividers.
- Level-3 prompt-style headings must remain visible as rhythm markers.
- Do not flatten everything into plain paragraphs.

## Cover rules

- Cover is independent from body images.
- Cover must not be generated by enlarging a body image.
- Cover must follow the local cover path from the package.
- WeChat main cover should be treated as a horizontal card asset, not a generic poster.
- Before replacing the cover, prefer verifying the local file is the intended `2.35:1` main cover asset.
- If the cover cannot be replaced safely, stop and report that cover replacement is still pending.

## Image placement rules

Push must follow `正文结构与配图映射.md`.

That means:

- image 01 is cover only
- image 02/03/04... each have a fixed insertion point
- each image must be inserted after the mapped heading/paragraph block
- the push step must not re-decide where the images “look better”

## Success conditions

The draft update is considered successful only when all of these are true:

1. title is visibly written
2. heading hierarchy is visibly preserved
3. body blocks are visibly inserted
4. cover preview shows the intended cover
5. mapped body images are near their intended explanation blocks
6. the flow reached the save-draft step
7. the cover visible in WeChat preview is the horizontal main cover rather than a cropped vertical poster
