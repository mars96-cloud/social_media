# 使用示例

## 只做配图规划

```text
Use $potato-illustrations 先不要生图。请分析下面这篇文章哪些段落值得配图，输出 5 张左右的 shot list。每张图写清楚：
- 放在哪个段落后
- 图的主题
- 核心意念
- 结构类型
- 土豆崽在图里做什么
- 建议元素
- 建议中文标注词
<粘贴文章>
```

## 直接生成正文配图

```text
Use $potato-illustrations 把下面这篇文章生成 4 张土豆崽正文配图。要求：16:9 横版、纯白背景、黑色手绘线稿、少量红橙蓝中文手写批注。每张图只讲一个核心结构，不要做 PPT 信息图，不要可爱卡通。
<粘贴文章>
```

## 长文配图策略

```text
Use $potato-illustrations 给这篇长文做配图策略。不要平均配图，只挑认知锚点：核心判断、输入输出闭环、前后对比、常见坑、承接路径。默认 6-8 张，先输出 shot list，不要生成图片。
<粘贴文章>
```

## 单个观点做一张图

```text
Use $potato-illustrations 为这个观点生成一张 16:9 正文配图：
信任不是喊出来的，而是一块证据一块证据铺过去。
画面要怪诞但清爽，土豆崽必须承担核心动作。中文标注最多 5 个，短一点。
```

## 工作流主题

```text
Use $potato-illustrations 为“把一条原始素材加工成流量、信任、转化三种内容”生成一张图。不要画正式流程图，不要复刻旧案例。请重新发明一个新的低科技隐喻，让土豆崽参与核心动作。
```

## 改图：去掉标题

```text
Use $potato-illustrations 帮我编辑这张图。去掉左上角的“流程图 / Workflow”标题和下划线，其他内容保持不变。不要新增任何文字或物件。
```

## 改图：增强土豆崽参与感

```text
Use $potato-illustrations 这张图方向对，但土豆崽有点像装饰。请保持核心意思不变，重生成一版：让土豆崽成为真正推动结构运转的主体。画面更怪一点，但仍然纯白、清爽、少字。
```
