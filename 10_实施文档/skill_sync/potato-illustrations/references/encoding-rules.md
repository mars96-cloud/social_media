# 防乱码规则

## 目标

以后这套 `potato-illustrations` skill 里的所有说明文件，必须稳定显示为简体中文，不能再出现乱码。

## 一劳永逸的处理方式

### 1. 统一编码

所有这些文件统一使用：

- UTF-8 编码

适用范围：

- `.md`
- `.yaml`
- `.json` 里的人类说明文本

### 2. 不在终端里拼长段中文

不要用 shell 字符串拼接、替换、重定向的方式去生成长篇中文说明。

因为很多乱码问题，本质上都是：

- 终端编码
- shell 默认编码
- 重定向写文件时的编码

三者不一致造成的。

### 3. 中文说明优先整文件重写

如果一个中文说明文件已经乱码，不要试图在乱码文本上局部修修补补。

正确做法：

- 直接整文件重写
- 保存为 UTF-8

这比在脏文本上继续补更稳。

### 4. skill 内部统一写简体中文

这套 skill 以后默认：

- 正文说明全部用简体中文
- 不混写英文说明段落

只有下面这些可以保留英文：

- YAML `name`
- 命令
- 路径
- 必要模型名
- 必要工具名

### 5. 中文路径要写真实路径

涉及角色锚点时，不要只写“参考土豆崽图片”这种抽象说法。

如果参考资产已经收进 skill 内部，优先写 skill 内部相对路径，比如：

- `assets/potato-ip/images/01_角色设定主图_定版.png`

这样即使后续再有显示问题，也不容易误解引用对象。

### 6. 后续新增文件时也必须遵守

以后只要在这个 skill 里新增：

- reference
- prompts
- examples
- 规则文档

都必须继续保持：

- 简体中文
- UTF-8
- 不通过 shell 拼长段中文
