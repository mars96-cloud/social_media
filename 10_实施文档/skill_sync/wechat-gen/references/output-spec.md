# 输出规格

## 目录结构

```text
<文章目录>/
  发布稿.md
  草稿箱正文.txt
  配图方案.md
  正文结构与配图映射.md
  封面方案.md
  metadata.json
  审核记录.md
  prompts/
  images/
```

## 文件要求

### 发布稿.md

包含：

- 标题
- 摘要
- 正文
- 建议插图位置
- 结尾 CTA

### 草稿箱正文.txt

要求：

- 纯文本
- 无 Markdown 标记
- 段落清晰
- 可直接供 `wechat-push-skill` 转写

### 配图方案.md

每张图必须写清：

- 图号
- 对应段落
- 这张图要表达什么
- 土豆崽动作与表情
- 背景与道具
- 配色
- 画面中文建议
- 是否建议后期叠字
- 实际 prompt 存放路径

### 正文结构与配图映射.md

必须写清：

- 一级标题
- 二级标题列表
- 三级标题列表
- 封面图文件
- 每张正文图对应的文件名
- 每张正文图插在第几个二级标题下
- 每张正文图插在该节第几个段落后

这是 `wechat-push-skill` 推送时的唯一插图执行依据，不能缺失。

### 封面方案.md

至少包含：

- 封面主标题
- 备选短标题
- 点击理由
- 画面主视觉
- 土豆崽表情
- 配色
- 文字叠加建议
- 如果出图，封面底图 prompt

### 封面成品

如果已经实际出图，封面目录里建议至少包含：

- `01_公众号封面_标题版_2.35比1.png`
- `01_公众号封面_标题版_1比1.png`
- `01_公众号封面_底图_2.35比1.jpeg`
- `01_公众号封面_底图_1比1.jpeg`

其中：

- `2.35比1`：公众号主文章卡、列表卡默认封面
- `1比1`：公众号主页、转发卡等方形场景
- 默认推荐像素：
  - `2.35比1` 主封面：`2820 x 1200`
  - `1比1` 方形封面：`1600 x 1600`
- 不允许只交竖版封面图
- 不允许让后台自动裁切竖图充当主封面

### metadata.json

必须包含字段：

```json
{
  "platform": "wechat",
  "title": "",
  "digest": "",
  "source_article": "",
  "cover_path": "images/01_公众号封面_标题版_2.35比1.png",
  "body_image_paths": [],
  "status": "draft-ready"
}
```

### 审核记录.md

必须写：

- 审核结论
- 各项检查结果
- 需要重做的问题

## 命名规范

图片命名：

- `01_公众号封面_标题版_2.35比1.png`
- `01_公众号封面_标题版_1比1.png`
- `01_公众号封面_底图_2.35比1.jpeg`
- `01_公众号封面_底图_1比1.jpeg`
- `02_正文配图_认知纠偏.jpeg`
- `03_正文配图_流程概念.jpeg`

Prompt 命名：

- `prompts/01_公众号封面.md`
- `prompts/02_正文配图_认知纠偏.md`

映射文件命名：

- `正文结构与配图映射.md`

## 入草稿兼容要求

要兼容 `wechat-push-skill`，必须满足：

- 有明确标题
- 有正文纯文本
- 有摘要
- 有封面路径
- 有正文图顺序
