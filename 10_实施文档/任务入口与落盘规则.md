# 任务入口与落盘规则

这份文档定义总控 skill 在工作区中的常用任务入口，以及每种任务应该把结果放到哪里。

## 任务入口

### 1. 写新内容

适用说法：

- 写一篇小红书
- 写一篇公众号
- 写一个抖音脚本
- 围绕某个主题出一条内容

默认动作：

1. 先输出内容决策卡
2. 再输出平台成稿
3. 再写入草稿区

默认落盘：

- `06_运营中心/<年份>/<月份>/草稿区`

### 2. 多平台改写

适用说法：

- 把这篇公众号改成小红书
- 把这条小红书改成公众号
- 把一个主题拆成多平台版本

默认动作：

1. 提炼母稿核心观点
2. 分平台改写
3. 每个平台单独保存

默认落盘：

- `06_运营中心/<年份>/<月份>/草稿区`

### 3. 发布前检查

适用说法：

- 帮我检查这篇能不能发
- 收一下口
- 看看标题、CTA、标签是否合理

默认动作：

1. 判断平台
2. 给出主要问题
3. 生成修订版
4. 输出可暂存版本

默认落盘：

- 原文件所在目录
- 如无原文件，落到 `草稿区`

### 4. 准备暂存发布

适用说法：

- 帮我暂存到小红书
- 帮我准备公众号草稿
- 帮我整理成可以发布的版本

默认动作：

1. 整理最终文案
2. 整理标题
3. 整理标签或导语
4. 整理配图说明
5. 进入暂存准备

默认落盘：

- `06_运营中心/<年份>/<月份>/已发布/<内容目录>/`
这里存放：
- `发布稿.md`
- `配图方案.md`

### 5. 草稿转已发布

适用说法：

- 这条已经发了，帮我归档
- 把这篇从草稿转到已发布

默认动作：

1. 按内容标题新建目录
2. 保存最终发布稿
3. 保存配图说明
4. 从草稿区移出或保留源稿标记

默认落盘：

- `06_运营中心/<年份>/<月份>/已发布/<内容目录>/`

### 6. 复盘回写

适用说法：

- 帮我复盘这条内容
- 总结这篇为什么有效
- 把经验沉淀到系统里

默认动作：

1. 提取表现结论
2. 提取结构经验
3. 提取可复用表达
4. 回写系统文档

默认落盘：

- `07_复盘总结`
- 如涉及长期方法论，再回写 `02_内容系统` 或 `12_平台执行框架`
