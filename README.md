# AI趣创社社媒工作区

这是一个可长期复用的中文社媒工作区，用来统一管理账号战略、内容生产、增长实验、素材资产、运营节奏和复盘沉淀。

这个工作区有两个目标：

1. 作为你自己长期运营的主工作台。
2. 后续裁剪成可复制给别人的知识库母版。

## 工作区定位

这里不是单纯的“写稿文件夹”，而是一套可沉淀、可迁移、可复制的社媒运营系统：

- 战略层：决定“写什么、给谁写、为什么写、怎么承接”
- 执行层：决定“不同平台怎么写得更好看、更好发、更能转化”
- 协作层：决定“如何研究、改写、暂存、发布与复盘”

## 顶层目录

- `00_收件箱`：临时输入、待整理内容
- `01_品牌系统`：账号定位、用户画像、内容边界、表达风格
- `02_内容系统`：内容支柱、选题规则、写作方法、发布逻辑
- `03_增长系统`：增长目标、流量入口、转化动作、实验计划
- `04_产品系统`：产品阶梯、交付方式、商业化路径
- `05_素材资产库`：案例、提示词、脚本、视觉素材、检查清单
- `06_运营中心`：月度执行、按平台拆分的草稿区、发布区、数据记录
- `07_复盘总结`：有效模式、失败模式、用户反馈、阶段复盘
- `08_模板库`：内容模板、复盘模板、产品模板
- `09_归档区`：历史材料、已停用模块
- `10_实施文档`：使用说明、迁移说明、操作规范
- `11_写作技能`：工作区自带的总控 skill
- `12_平台执行框架`：小红书、公众号、抖音等平台执行规范

## 当前核心能力

### 1. 战略资料集中管理

工作区已经按“品牌 - 内容 - 增长 - 产品”拆好结构，后续所有内容生产都会先参考这些文档，而不是临时凭感觉写。

### 2. 总控写作 skill

内置 `social-media-workspace-operator`，作为统一入口管理：

- 内容战略判断
- 平台路由
- 多平台改写
- 草稿暂存
- 发布前检查

你后续不需要在 prompt 里手动写一串“先用哪个 skill，再用哪个 skill”，直接调用这一个入口即可。

### 3. 平台执行框架

`12_平台执行框架` 单独维护不同平台的写法、排版、配图、标题和检查规则。

这样做的好处是：

- 战略层和执行层分离，便于维护
- 后续复制给别人时，可以按需保留或删除生产层模块
- 新增平台时，只需要扩展执行框架，不必重写整个工作区

## 推荐使用方式

### 日常生产

1. 在 `00_收件箱` 放入灵感、资料、对话、素材。
2. 在 `06_运营中心` 推进当月选题、草稿、发布和记录。
3. 用总控 skill 按工作区规则生成平台内容。
4. 发布后把结果和反馈回写到 `07_复盘总结`。

### 迁移为知识库母版

第二阶段如果要复制给别人，通常保留：

- `01_品牌系统`
- `02_内容系统`
- `03_增长系统`
- `04_产品系统`
- `08_模板库`
- `10_实施文档`
- `11_写作技能`
- `12_平台执行框架`

按需移除：

- `06_运营中心`
- `07_复盘总结`
- 与个人日常生产强绑定的素材或草稿

## 典型用法

你后续可以直接这样提需求：

- “按 `social-media-workspace-operator` 来，写一篇公众号文章，主题是……”
- “按 `social-media-workspace-operator` 来，把这篇公众号改成小红书图文。”
- “按 `social-media-workspace-operator` 来，给这条内容做发布前检查，并整理成可暂存版本。”

## 维护原则

- 长期不变的方法论，写入系统层文档。
- 当月正在执行的内容，放在运营层。
- 复盘出来的规律，再回写到系统层。
- 总控 skill 负责统一调度，平台规则放在执行框架中独立维护。

## 下一步建议

如果继续完善，可以按这个顺序推进：

1. 继续补充账号定位、商业化、选题和转化文档。
2. 按平台积累更多“可直接发”的模板。
3. 把已验证有效的内容流程沉淀成固定 SOP。
4. 定期同步工作区 skill 到全局 skill，并推送到 GitHub。
