# 公众号执行规则

## 适用场景

用于公众号长文内容的最终执行层处理。

重点不是决定写什么，而是决定怎么写得更适合公众号阅读和信任沉淀。

## 标题规则

- 标题要清楚，不要玩太虚
- 标题优先体现问题、方法、路径、对象
- 不必像小红书那样强冲突，但必须一眼看懂

## 导语规则

- 开头先点出现实问题
- 再解释为什么值得讲
- 让读者知道这篇文章会解决什么

## 正文结构规则

- 用分节结构写
- 每节只讲一个核心点
- 先讲问题，再讲方法或判断
- 多用“为什么”“怎么做”“适合谁”“常见误区”这类结构

## 语言风格

- 更完整
- 更有解释感
- 更适合沉淀方法论
- 不要写成工具说明书

## 结尾规则

- 收束全文结论
- 告诉读者下一步可以做什么
- 和整个账号主线保持一致
- 只保留一个主动作，不要再写“回复 XXX 领取资料 / SOP / 清单”
- 当前账号默认改成关注留存型承接，引导读者持续看日常推送与直播
- 结尾动作要服务后续转化，重点是关注、点击、留存，不是一次性资料领取

## 推荐 CTA

公众号优先承接：

1. 关注我，后续日常推送与直播带你手把手开发自己的工作流与智能体。
2. 关注后续系列内容与案例拆解。
3. 为后续直播、训练营或低客单产品做预热。

## 排版建议

- 小标题清晰
- 每段不要过长
- 保持呼吸感
- 重点句可以单独成段

## 执行层硬规则

- 标题写入前先去掉 BOM 等不可见脏字符
- 正文图片先按正文顺序反向上传，再按结构块重新插入
- 封面必须单独走图片库选择流程，不要默认拿正文第一张图做封面
- 封面流程固定为：选择图片 -> 下一步 -> 编辑封面 -> 确认 -> 保存为草稿
- 自动化执行里，正文成功、封面成功、保存按钮可见，三者缺一不可
- “已保存 / 手动保存”提示不是唯一成功信号，不能把它当成唯一判定条件
- 只要封面预览真实挂上、流程进入 `save-draft`、保存按钮已执行点击，就视为草稿保存链路已完成
