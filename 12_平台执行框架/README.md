# 平台执行框架

这里是当前工作区的执行层规则包。

它和战略层分开：

- 战略层回答：写什么、为什么写、写给谁、承接什么
- 执行层回答：怎么写得更适合平台、怎么排版、怎么配图、怎么发前检查

## 使用方式

总控 skill 先做战略判断，再按平台读取这里对应的规则文件。

## 和工作流的关系

这一层只回答一件事：

- 内容已经确定要写了，接下来怎样写得更符合平台

它不负责决定：

- 这条内容该不该写
- 应该服务谁
- 当前阶段目标是什么
- 应该保存到哪里

这些由工作区战略层和总控 skill 中的 workflow 文档决定。

## 本次新增硬规则

- 公众号和小红书的封面 / 配图不能再默认产出同一视觉模板
- 批量生成时必须执行固定风格轮换，并把主题名称写回本地产物
- 公众号默认 CTA 已改为关注留存型承接，不再默认使用“回复关键词领资料 / SOP / 清单”
- 小红书默认把核心信息前置到图片页，不再把长段解释堆回正文
- 公众号封面必须单独选择并确认，不能拿正文首图兜底
- 公众号方法型文章除封面外，默认至少准备 5 到 8 张文中理解型配图
- 公众号小标题必须在转换和落稿时保留为独立段，不能并回正文
