# Social Media Workspace Operator

## 概述

这是当前社媒工作区的总控 skill。

它不是单个平台写手，而是一个统一入口，负责把工作区里的战略文档、平台执行框架、运营目录和发布准备动作串起来。

它默认管理六类任务：

1. 新内容写作
2. 多平台改写
3. 发布前检查
4. 暂存发布准备
5. 草稿转已发布
6. 复盘回写

## 什么时候使用

在下面这些情况使用这个 skill：

- 要写一篇小红书、公众号或抖音脚本
- 要把一条内容改成多个平台版本
- 要按当前工作区规则出稿
- 要准备草稿暂存或发布前资产
- 要把草稿整理为已发布归档结构
- 要对内容做复盘并回写方法论

不要在下面这些情况使用：

- 只是单纯整理目录
- 只是查看数据，不涉及内容产出
- 只是修改无关项目文件

## 核心原则

1. 先战略判断，再平台执行。
2. 先产生成稿，再进入发布准备。
3. 先落盘到工作区，再进入协作动作。
4. 默认只到草稿、暂存或预发布准备阶段，不自动终发布。
5. 用户不需要手动点名多个 skill，总控 skill 自己负责路由。

## 工作流

### 第一步：识别任务类型

先判断当前任务属于：

- 新写一条
- 改写
- 检查
- 暂存发布
- 归档
- 复盘

如果任务类型不明确，先用用户原话和任务目标补判断；仍不明确再提问。

### 第二步：按顺序读取工作区

严格优先读取：

1. `01_品牌系统`
2. `02_内容系统`
3. `03_增长系统`
4. `04_产品系统`
5. `06_运营中心/当月`
6. 历史草稿和已发布内容
7. 目标平台执行规则

不要无差别扫完整个工程。

### 第三步：先做内容决策

每次写作或改写前，都先确认四件事：

- 目标人群
- 内容支柱
- 当前阶段目标
- 主 CTA

如果这四项里有关键项不清楚，不要直接硬写。

### 第四步：按平台执行

根据平台输出对应格式：

- 小红书：标题候选、开头钩子、正文、CTA、标签
- 公众号：标题、导语、分节正文、结尾、CTA
- 抖音：主题方向、3 秒钩子、口播脚本、镜头提示、CTA

### 第五步：落盘到工作区

如果任务涉及产物文件，按下面规则处理：

- 新内容或改写稿：默认进入 `06_运营中心/<年份>/<月份>/草稿区`
- 准备发布或归档：整理为 `06_运营中心/<年份>/<月份>/已发布/<目录名>/`
- 复盘结论：进入 `07_复盘总结`
- 长期方法论：回写 `02_内容系统` 或 `12_平台执行框架`

### 第六步：协作与发布准备

如果任务包含“暂存、发到平台、准备发布”：

1. 先准备最终文案
2. 再准备标题、标签、导语或配图说明
3. 再进入发布准备流程

默认只进入草稿或暂存环节，不自动终发布。

## 默认输出顺序

除非用户明确只要最终稿，否则默认输出：

1. 内容决策卡
2. 平台成稿
3. 如有需要，再补充落盘结果、暂存准备或回写建议

## 用户入口合同

用户以后可以直接这样说：

- “按 `social-media-workspace-operator` 来，写一篇公众号，主题是……”
- “按 `social-media-workspace-operator` 来，把这篇公众号改成小红书和抖音脚本。”
- “按 `social-media-workspace-operator` 来，检查这篇内容能不能发。”
- “按 `social-media-workspace-operator` 来，整理成可暂存到小红书的版本。”
- “按 `social-media-workspace-operator` 来，把这条内容从草稿转成已发布归档。”

用户不需要手动说明：

- 去读哪个 md
- 用哪个 skill
- 存到哪个目录
- 哪一步先做

这些都由总控 skill 自己处理。

## 常见错误

### 错误 1：跳过战略判断，直接开写

修正：

先输出内容决策卡，再进入平台成稿。

### 错误 2：只生成对话结果，不回写工作区

修正：

涉及正式产物时，必须有明确落盘意识。

### 错误 3：把战略层和执行层混在一起

修正：

- 战略层由 `01_品牌系统` 到 `04_产品系统` 决定
- 执行层由 `12_平台执行框架` 决定

### 错误 4：进入发布动作前还没有最终成稿

修正：

必须先准备好最终文本和平台必需资产，再进入暂存或发布准备。

### 错误 5：让用户自己在 prompt 里编排多个 skill

修正：

总控 skill 自己负责内部路由，不把调度责任抛回给用户。

## 合格结果标准

如果这个 skill 输出合格，应该满足：

- 看得出内容符合账号定位
- 看得出平台差异而不是一稿硬套
- 有清晰内容决策卡
- 有明确主 CTA
- 如有需要，知道该保存到哪里
- 如进入发布流程，默认停在草稿或暂存阶段
