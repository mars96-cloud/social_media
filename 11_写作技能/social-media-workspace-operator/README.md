# social-media-workspace-operator

这是当前社媒工作区的总控 skill。

它的职责不是单纯“帮你写一篇内容”，而是把工作区里的战略资料、平台执行规则和协作发布动作串成一个统一入口。

## 适用场景

适合处理这些任务：

- 写小红书图文
- 写公众号文章
- 写抖音口播稿或视频脚本
- 把一条内容改成多平台版本
- 做发布前检查
- 准备草稿暂存或协作发布材料

## 三层结构

### 1. 战略判断层

先判断这条内容是否符合：

- 账号定位
- 目标人群
- 当前阶段目标
- 内容支柱
- 承接动作

### 2. 平台执行层

再根据目标平台调用执行框架，分别处理：

- 小红书标题、首图、正文、标签
- 公众号标题、导语、分节、排版
- 抖音钩子、口播节奏、镜头结构

### 3. 协作发布层

当任务包含“暂存、发布、改写、协作”时，再进入对应流程。

默认只走到草稿和预发布准备阶段，不自动终发布。

## 依赖关系

这个 skill 会优先读取工作区内的系统文档，并参考：

- `11_写作技能/social-media-workspace-operator/references/`
- `12_平台执行框架/`

如果环境中已经安装相关能力，也会按需借用对应能力做研究、改写或发布准备，但用户不需要在 prompt 里手动逐个点名。

## 推荐用法

直接这样说即可：

- “按 `social-media-workspace-operator` 来，写一篇公众号文章，主题是……”
- “按 `social-media-workspace-operator` 来，把这篇文章改成小红书版本。”
- “按 `social-media-workspace-operator` 来，给这条内容做发布前检查，并准备暂存。”

## 维护方式

- 工作区版本：作为长期可维护源码
- 全局版本：作为本机安装版
- GitHub 项目仓库：管理整个工作区
- GitHub skill 仓库：管理全局可复用 skill
