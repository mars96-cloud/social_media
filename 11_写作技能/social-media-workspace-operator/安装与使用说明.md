# 安装与使用说明

## 当前定位

这个 skill 的源码主版本放在当前工作区中，适合：

- 跟随整个项目一起备份到 GitHub
- 在工作区内持续迭代规则
- 稳定后再同步到本机全局 skill

## 本地安装路径

如果要让 `Codex` 全局自动发现它，把整个目录复制到：

`C:\Users\Administrator\.codex\skills\social-media-workspace-operator`

建议复制的内容包括：

- `SKILL.md`
- `README.md`
- `安装与使用说明.md`
- `references/`
- `examples/`

## 推荐调用方式

以后不需要再手写一长串 prompt，也不需要手动写：

- 先用哪个 skill
- 再用哪个 skill
- 最后用哪个 skill

直接调用这一个总控入口即可，例如：

- “按 `social-media-workspace-operator` 来，写一篇小红书图文，主题是……”
- “按 `social-media-workspace-operator` 来，写一篇公众号文章，主题是……”
- “按 `social-media-workspace-operator` 来，把这条内容改成多平台版本。”
- “按 `social-media-workspace-operator` 来，做发布前检查，并准备草稿暂存。”

## 它会自动做什么

这个 skill 会按顺序完成：

1. 读取工作区中的账号定位、内容系统、增长目标等基础资料。
2. 先给出内容决策判断，而不是直接开写。
3. 判断应该输出哪个平台版本。
4. 参考 `12_平台执行框架` 优化格式、结构、表达和排版。
5. 如果任务涉及协作或发布，继续进入暂存与发布前准备流程。

## 默认边界

- 默认停在“草稿 / 暂存 / 预发布准备”阶段。
- 不会在未明确确认的情况下直接终发布。
- 如果账号定位、人群、目标、CTA 不清楚，会先补读工作区文档或提出澄清。

## 后续扩展方向

- 增加更多平台规则
- 增加配图和封面生成联动
- 增加已发布内容回读与复盘联动
- 增加更细的批量改写与批量暂存流程
