# 调用示例

## 示例 1：写小红书

```md
按 social-media-workspace-operator 来。

先按工作区读取顺序读取需要的 md。

帮我写一篇小红书：
主题：普通人学 AI，到底先提效还是先变现

要求：
- 先给我内容决策卡
- 更偏小红书表达
- 强钩子
- 不空
- 主 CTA 用私信承接
```

## 示例 2：写公众号

```md
按 social-media-workspace-operator 来。

先读取品牌系统、内容系统、当月执行包，再写。

帮我写一篇公众号：
主题：从 AI 提效到 AI 变现，普通人更现实的一条路

要求：
- 先判断这篇更适合哪个人群和当前目标
- 逻辑完整
- 不做工具盘点
- 结尾承接资料包
```

## 示例 3：改成多平台

```md
按 social-media-workspace-operator 来。

先读取工作区规则。

把“普通人最值得先搭的 3 条 AI 工作流”这个主题分别改成：
- 小红书版
- 公众号版
- 抖音脚本版

先输出统一的内容决策卡，再分平台改写。
```

## 示例 4：发前收口

```md
按 social-media-workspace-operator 来。

先判断这是小红书稿。

帮我做发前检查：
- 标题再收一下
- CTA 不要太散
- 改成更适合收藏和评论的版本
```
