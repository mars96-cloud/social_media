# 协作技能映射

## 目标

这个总控 skill 不是要替代所有已安装 skill，而是统一调度它们。

用户只需要记住一个入口：

- `social-media-workspace-operator`

然后由总控 skill 内部判断什么时候借用其他已安装 skill 的能力。

## 推荐映射

### `social-media-context-sms`

适用：

- 账号身份、语气、人群信息缺失
- 需要补全或更新 `.agents/social-media-context-sms.md`

总控 skill 的职责：

- 如果上下文不完整，先提示补齐
- 不要在缺定位的情况下直接强写

### `content-pipeline`

适用：

- 用户要一整套内容流程
- 用户要公众号 + 小红书联动
- 用户要从选题一直走到暂存发布

总控 skill 的职责：

- 采用 staged workflow
- 先产出平台内容资产，再进入发布准备

### `post-writer-sms`

适用：

- 平台和主题都已经明确
- 需要快速出单平台草稿

总控 skill 的职责：

- 在战略判断完成后，单平台写作可参考它的结构思路
- 但输出仍要服从本工作区战略层和执行层

### `content-repurposer-sms`

适用：

- 需要一稿多发
- 需要长文改图文、图文改脚本

总控 skill 的职责：

- 多平台改写时采用“先提主逻辑，再拆平台版本”的方式

### `platform-strategy-sms`

适用：

- 平台打法本身不明确
- 用户在问某个平台该怎么做

总控 skill 的职责：

- 平台策略未定时，先补平台判断，再进入内容写作

### `xhs-search` / `xhs-content-plan`

适用：

- 做小红书选题研究
- 看热门内容、竞品、爆款方向

总控 skill 的职责：

- 需要研究时优先借这类能力
- 再回到工作区做选题判断

### `social-push`

适用：

- 用户要求把内容发到平台
- 用户要求上传文章、图文、视频

总控 skill 的职责：

- 统一由总控 skill 准备发布资产
- 真正浏览器暂存发布动作交给 `social-push`
- 默认只能暂存草稿，不能自动终发

## 对用户的承诺

用户不需要自己写：

- 用哪个写作 skill
- 用哪个改写 skill
- 用哪个发布 skill

总控 skill 应该自己判断任务属于：

- 战略判断
- 单平台写作
- 多平台改写
- 发布暂存

再内部路由。
