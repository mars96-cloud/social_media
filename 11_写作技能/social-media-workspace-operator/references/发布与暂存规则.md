# 发布与暂存规则

## 核心原则

总控 skill 负责内容准备与平台判断，真正的浏览器发布动作统一交给 `social-push`。

## 默认允许的动作

- 暂存草稿
- 进入预览
- 准备待发布内容

## 默认禁止的动作

- 自动点击最终发布
- 用户未确认时自动终发
- 在内容和素材未准备好的情况下直接推送

## 进入发布前，总控 skill 必须准备好

1. 目标平台
2. 内容类型
3. 最终文案
4. 标题
5. 标签
6. 图片或视频素材说明
7. 是否只存草稿

## 各平台处理

### 小红书

- 先准备最终图文稿
- 再准备首图文案、标签、卡片结构说明
- 再交给 `social-push` 走小红书 workflow

### 微信公众号

- 先准备最终长文稿
- 再准备标题、导语、正文结构
- 再交给 `social-push` 走微信公众号文章 workflow

### 抖音

- 当前优先支持脚本和发布前准备
- 真正视频上传 workflow 后续再接

## 用户的统一说法

以后用户可以直接说：

- “按总控 skill 写完并暂存到小红书”
- “按总控 skill 写完并暂存到公众号”

总控 skill 要自己理解为：

1. 先做战略判断
2. 再做平台成稿
3. 再准备发布资产
4. 最后进入 `social-push` 暂存流程
