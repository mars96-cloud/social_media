# 统一入口提示词合同

这个 skill 的目标，是让用户只用一个入口就能驱动整个社媒工作区。

## 用户不需要说的内容

用户不需要手动写：

- 先用哪个 skill
- 再用哪个 skill
- 去读哪个 md
- 去哪个目录保存

这些动作都应由总控 skill 自己完成。

## 用户可以直接说的内容

### 新写一条

- 按 `social-media-workspace-operator` 来，写一篇公众号，主题是……
- 按 `social-media-workspace-operator` 来，写一条小红书，主题是……

### 多平台改写

- 按 `social-media-workspace-operator` 来，把这篇公众号改成小红书和抖音脚本

### 发布前检查

- 按 `social-media-workspace-operator` 来，检查这篇内容能不能发，并收口

### 暂存发布

- 按 `social-media-workspace-operator` 来，整理成可暂存到小红书的版本

### 归档复盘

- 按 `social-media-workspace-operator` 来，把这条内容从草稿转到已发布，并生成配图方案

## 总控 skill 的默认理解

收到任务后，默认执行：

1. 判断任务类型
2. 按读取顺序读取工作区
3. 先输出内容决策卡
4. 再输出平台成稿
5. 如有需要，补充落盘、暂存、复盘动作
