# 产物落盘与回写规则

总控 skill 不能只输出对话结果，必须有明确的工作区落盘意识。

## 基本原则

1. 新内容默认写入草稿区。
2. 准备发布的内容默认整理为已发布目录结构。
3. 复盘结果默认写入复盘层或系统层。
4. 没有明确落盘位置时，优先放到当月运营目录。

## 草稿区落盘

适用于：

- 首次成稿
- 多平台改写后的待审版本
- 发布前修订版本

目录：

- `06_运营中心/<年份>/<月份>/草稿区`

命名：

- `YYYY-MM-DD_平台_主题_草稿.md`

## 已发布结构落盘

适用于：

- 已定稿
- 准备暂存
- 已发布归档

目录结构：

- `06_运营中心/<年份>/<月份>/已发布/<YYYYMMDD_平台_主题>/发布稿.md`
- `06_运营中心/<年份>/<月份>/已发布/<YYYYMMDD_平台_主题>/配图方案.md`

## 回写系统层

当出现以下情况时，应该回写系统层：

- 新发现一个稳定有效的内容结构
- 新发现一个平台表达规律
- 新发现一个转化动作更有效
- 新发现一个高频失败点值得规避

回写位置：

- 战略类：`01_品牌系统` 到 `04_产品系统`
- 内容方法类：`02_内容系统`
- 执行类：`12_平台执行框架`
- 复盘类：`07_复盘总结`

## 用户交付口径

如果任务包含文件生产，总控 skill 应明确说明：

- 本次生成了什么
- 放到了哪里
- 下一步应该做什么
