# 配图方案

## 配图总原则

- 公众号是文字为主，配图为辅
- 主角固定使用土豆崽 IP
- 风格固定参考 `potato-illustrations`
- 呈现方式参考 `redbook-gen` 的拆解结构
- 图中文字尽量少，优先后期本地叠字
- 每张图只解释一个关键判断
- 不做小红书漫画卡页，不做密集信息图，不做海报
- 全文配图统一横版

## 02 正文配图：一直在点状使用

- 对应段落：
  “很多人现在用 AI，还是一种‘点状使用’……这些动作大多是零散的”
- 本图观点：
  工具越攒越多，但始终没有连成流程
- 图中可见短文案：
  一直在点着用
- 画面内容：
  土豆崽陷在一堆零散工具、聊天气泡、提示词纸片里，前面是一条断掉的路径线
- 土豆崽动作与表情：
  忙乱、无奈、手上还抓着纸片
- 背景与道具：
  白底；小便签、聊天框、断箭头、小齿轮
- 配色：
  土豆黄 + 黑线稿 + 浅蓝橙红点缀
- 构图方向：
  横版，土豆崽偏左中，右边留给断开的路径
- Prompt 路径：
  `prompts/02_正文配图_点状使用.md`
- 审核要点：
  不能太花，不能像海报，土豆崽必须是主体

## 03 正文配图：工作流是一条能跑通的路

- 对应段落：
  “什么叫工作流……拆成几个固定环节”
- 本图观点：
  工具解决的是点，流程解决的是结果
- 图中可见短文案：
  先跑通一条
- 画面内容：
  土豆崽站在一条清晰四节点流程旁，乱的部分被收束成一条路
- 土豆崽动作与表情：
  恍然大悟，视线指向路径
- 背景与道具：
  白底；四个简化节点；两个小 AI 辅助图标
- 配色：
  土豆黄 + 浅蓝路径 + 橙色强调点
- 构图方向：
  横版，路径从左到右延展
- Prompt 路径：
  `prompts/03_正文配图_工作流路径.md`
- 审核要点：
  不能做成复杂流程图，不能堆太多节点

## 04 正文配图：先问这 3 个问题

- 对应段落：
  “你可以先问自己三个问题……”
- 本图观点：
  普通人先找高频卡点，再搭第一条流程
- 图中可见短文案：
  先找卡点
- 画面内容：
  土豆崽坐在桌前盯着三张问题卡，一张卡被重点圈出来
- 土豆崽动作与表情：
  认真、专注、在思考
- 背景与道具：
  白底；三张卡片；重点圈选标记
- 配色：
  土豆黄 + 米白卡片 + 红橙强调圈
- 构图方向：
  横版，主体居中偏左
- Prompt 路径：
  `prompts/04_正文配图_三个问题.md`
- 审核要点：
  图不能太空，卡片不能太多，重点必须一眼可见
