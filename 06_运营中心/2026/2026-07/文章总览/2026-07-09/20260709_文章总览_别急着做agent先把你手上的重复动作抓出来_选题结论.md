# 选题结论

## 选题

别急着做 agent，先把你手上的重复动作抓出来

## 选题结论

Agent 真正有价值的前提，是你已经有稳定、重复、可描述的动作；否则它只是在自动化混乱。

## 目标读者

被 agent、智能体概念吸引，但实际业务还没跑顺的人。

## 写作目标

避免用户跑偏到炫技术，继续守住账号定位。

## 内容边界

重点写 agent 适用前提，不写技术教程和部署细节。

## 建议大纲

1. 为什么很多人一听 agent 就兴奋
2. 什么时候 agent 才有意义
3. 重复动作都没固定，agent 也帮不了你
4. 普通人现在更该先做的 3 件事
5. agent 是下一阶段升级，不是第一步

## 表达方式

口语化、判断型、教练型。先指出误区，再给最小起步动作，不写成 AI 行业分析。

## 承接动作

承接工作流升级模块。
