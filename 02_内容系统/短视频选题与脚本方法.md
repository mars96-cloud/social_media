# 短视频选题与脚本方法

## 适用范围

这套方法只用于 20-30 秒短视频与短剧化表达，不用于公众号文章结构。

## 选题优先级

优先选择以下题材：

1. 高频低效问题
2. 常见误区
3. 前后对比明显的工作流改造
4. 可用 1 个动作解释清楚的技巧
5. 能自然承接到评论、私信或长文的主题

## 默认脚本结构

### 结构一：问题 -> 原因 -> 解法

- 前 3 秒：抛出痛点
- 中段：指出常见错误
- 结尾：给出更优动作和 CTA

### 结构二：错误做法 -> 正确做法

- 前 3 秒：指出常见错误
- 中段：展示替代流程
- 结尾：给出结果或下一步

### 结构三：前后对比

- 前 3 秒：直接给反差
- 中段：拆出关键差异
- 结尾：推动收藏、评论或跳转

## 分镜拆分原则

- 20-30 秒控制在 6-8 个镜头更稳。
- 每个镜头只承载 1 个主要信息点。
- 先有分镜图，再做镜头视频。
- 每个镜头都必须能单独审核，不依赖“脑补下一镜”。

## 图片阶段检查

- 主体是否明确
- 角色是否一致
- 视角是否符合脚本
- 背景是否干净
- 是否有不该出现的文字
- 是否具备视频延展空间

## 视频阶段检查

- 动作是否自然
- 镜头运动是否稳定
- 口播和字幕是否能对齐
- 是否需要补反应镜头或空镜
- 是否具备剪辑价值

## 短视频默认 CTA

- 评论区领取
- 收藏备用
- 私信关键词
- 去看长文版
- 看下一条系列内容
